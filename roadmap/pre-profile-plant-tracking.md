# 🌱 Feature Addition: Pre-Profile Creation & Plant Tracking System

We need to introduce a **Pre-Profile Creation** feature for individuals who are unable to create an account themselves (e.g., children, elderly people, laborers, uneducated individuals, or those with limited access to technology).  

This will also help in situations where there is not enough time to guide them through the registration process during sapling distribution.

---

# 1️⃣ Pre-Profile Creation (By Authorized Team Members)

## Purpose
Enable authorized NGO team members to manually create temporary profiles for sapling recipients.

## Access Control
- Only **authorized and trusted NGO team members** should have access.
- Access provided through a **restricted form screen** within the app.

## Required Form Fields
The form should capture the following essential information:

- Full Name  
- Phone Number (**MANDATORY**)  
- Email ID (**MANDATORY**)  
- Age / Date of Birth  
- Location (with map pin option if possible)  
- Plant Species Gifted  
- Any additional necessary details  

## Unique Linking Requirements
Each pre-profile must be uniquely linked to:
- Phone Number  
- Email ID  
- Name  

⚠️ It is mandatory to collect accurate phone number and email address to ensure proper future linking.

## Outcome
- A temporary/pre-profile is created.
- The profile is directly linked to the gifted plant.

---

# 2️⃣ Account Linking

## Automatic Linking
- If the recipient later creates an account using the same **phone number and/or email ID**,  
  the previously recorded plant data should automatically link to their account.

## Post-Link Behavior
- The plant should appear in the user’s profile as one recorded plantation.
- No manual intervention should be required.

---

# 3️⃣ Plant Photo Upload & Growth Updates

## Initial Upload Requirement
- After planting the sapling, the user must upload a photo of the planted sapling.

## Growth Tracking
- Users should be able to post regular updates about:
  - Plant growth
  - Health condition
  - Maintenance activities

## Timeline Feature
- All updates should appear in a **timeline view** under that specific plant.

---

# 4️⃣ Community Interaction & Map Visibility

## Public Map View
- Each planted sapling should appear on a **public map**.
- Location should be based on the recorded map pin.

## Community Features
Community members should be able to:
- View plant details  
- Like updates  
- Comment  
- Share suggestions or feedback  

## Purpose
- Encourage transparency  
- Promote accountability  
- Strengthen community engagement  

---

# 5️⃣ Care Reminders & Responsibility Tracking

## Automated Reminders
- If no updates are received for a certain period:
  - The system should send automated reminders to the plant owner.

## Volunteer Notifications
- If a plant is marked as:
  - “No caretaker”
  - Inactive
- The system should notify nearby volunteers or community members.

## Goal
Ensure that no planted sapling is neglected.

---

# 6️⃣ Data Management

## In-App Data View
- All collected data should be displayed in a structured list format within the app.

## Export Functionality
- Data must be exportable as:
  - Excel (.xlsx) file
- For reporting and documentation purposes.

---

# ✅ System Objectives

This complete system ensures:

- No sapling goes unrecorded  
- Every plant is traceable  
- Community engagement is encouraged  
- Accountability and long-term care are maintained
